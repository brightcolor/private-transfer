# Contributing

Use small pull requests with focused changes. Keep code simple, avoid duplicate logic, and prefer existing Laravel conventions before introducing new patterns.

## Workflow

1. Create a branch from `main`.
2. Use Conventional Commits.
3. Add or update tests for behavior changes.
4. Run `composer test` and `npm run build`.
5. Update `CHANGELOG.md` for user-visible changes.

No secrets, production data, or private credentials should be committed.
